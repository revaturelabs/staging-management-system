Summary of Project

The Staging Management System application is designed to allow end users to Add/View/Update current and future human resources. It is intended to replace the former Excel sheet and additional features.

Summary of Methodology

What general development approach will be used?
Agile Scrum Development: It is a lightweight project management framework with a broad applicability used to aid in managing and controlling iterative/incremental application development. For Scrum, the "Product Owner" works closely with a development team to identify and prioritize application functionalities in the form of a "Product Backlog" which contain user stories; For example: "As a user, I can log in". This product Backlog contains application features, bug fixes, and non-functional requirements. Developers take these user stories and undergo a sprint in which they create an incremental build of the application which lowly takes shape into the final build over multiple sprints. As stated in the Software Development Methodology.
What development and collaboration tools will be use?
We plan to use the following tools extensively through out the project:
  • Spring Tool Suite
  • Maven
  • Git
  • Hansoft
  • Jenkins
  • Slack
  • Apache Tomcat
  • Log4J
How will changes be controlled?
  • Requests for merges will be controlled by 2 team members who will ensure version integrity
  • All future source code commit log messages must refer to a specific issue ID.
How will this plan be updated?
This project plan will be updated as needed throughout the project. It will be placed under version control and instructions for accessing it will be on the project website.
