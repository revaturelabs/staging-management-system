Background and Motivation

What is the setting and history behind this project?
Originally, human resources such as associates have been mapped exclusively by an Excel sheet. The data therein only consisted of those who actively checked in daily and had no way of fortelling how many associates would be available in future weeks. The idea was conceived around May 10th, 2017 when the Product-Owner requested a way to revolutionize the current system.
What is the problem to be addressed?
With the idea of overhauling the current system, the web based application should be capable of creating and tracking associates throughout the Revature Program with the functionality of pairing Associates with Client Companies. By adding this functionality, the use of the prior Excel sheet can be replaced and allows the creation of added functionalities.
What are some current approaches to this problem?
The end user can log associates into an excel sheet to record daily attendance. Later, this prior functionality was built upon by allowing associates to log in daily in order to record attendance.
Why is this problem worth solving or worth solving better?
At the prior stages there were no way to properly track Associate pairings with Client Companies, which in turn would create problems if an associate was placed with a different Client Company if they were already reserved for another.
How will this product be better than previous approaches?
Our approach is aimed at providing a relevant and interactive view that can aid in dynamic allocation of human resources. In development, the main focus was to be able to view the total number of current and future associates; while providing a way to map them to clients if they are available.
Our system will have enhanced functionality, better maintainability, scalability, and security.

Goal

What is the goal of this project?
This project will produce an interactive web application that allows end users to create/view/update associate, batch, and client information
What are the defining features and benefits of this product?
  • Reusable web application with functionality for creating, editing, deleting, searching, and assigning associates/batches/clients. This maintains and refreshes to ensure that users will always find information that is automatically up-to-date.
  • The web application will be secure and only allow users with the proper permissions access. This will prevent the addition, modification, and removal of information.

Scope

We want to focus on the web application itself, and the features of that application that help create a means of providing a coherent view of current and future associate data.

  • Work with Apache Tomcat
  • Develop with Spring and Hibernate
  • The web site provides ways for a user to change the status and linking of an Associate to a Client.

Risks and Rewards

What are the main risks of this project?
  1. There are significant technical difficulties in building a web application using the advanced technologies such as: AngularJS, Spring, Hibernate, etc.. This will be a risk because it is an initial approach to AngularJS development. We will certainly make some mistakes and suboptimal choices. However that may be, we will address this risk by scoping the project such that we have enough time to train and to review the design and implementation.
  2. The schedule for this project is very short. We will manage this by planning a conservatively scoped functional core and series of incremental builds in which can implement added functionalities for future iterations..
What are the main rewards if this project succeeds?
If we accomplish the elements of our plan, our Staging Management System will replace the relic Excel sheet and provide a more modern approach to the problem.
